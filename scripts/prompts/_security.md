## セキュリティルール（全ロール共通・厳守）

### 操作範囲の制限

- 操作対象はプロジェクトフォルダ（カレントディレクトリ）配下のみとする
- プロジェクトフォルダ外のファイル・ディレクトリの読み取り・書き込み・削除は一切禁止
- `cd` でプロジェクト外に移動してはならない
- 絶対パスを使う場合はプロジェクトルート配下であることを確認すること

### Git 操作の制限

以下のファイルは絶対に `git add` / `git commit` しないこと:

- `.env`, `.env.*` — 環境変数
- `*.pem`, `*.key`, `*.crt`, `*.jks`, `*.p12`, `*.keystore` — 証明書・秘密鍵
- `*secret*`, `*credential*`, `*token*` — 秘匿情報を含むファイル
- `tasks/logs/` — タスク実行ログ
- `tasks/.runner.pid` — PID ファイル
- `*.log` — ログファイル

コミット前に `git status` で対象ファイルを確認し、上記に該当するものが含まれていないことを検証すること。

### ログ・出力の制限

- API キー、トークン、パスワード、秘密鍵の内容をコンソール出力やログに書き出さない
- `.env` ファイルの内容を表示・出力しない
- エラーメッセージに秘匿情報が含まれる場合はマスクすること

### 禁止操作

- `git push` は行わない（人間が確認して実行する）
- `git reset --hard`, `git clean -f` などの破壊的操作は行わない
- `rm -rf` を広範囲に実行しない（個別ファイルの削除は可）
- 外部への HTTP リクエスト（curl, wget）で秘匿情報を送信しない
