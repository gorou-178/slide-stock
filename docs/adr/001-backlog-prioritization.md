# ADR-001: バックログタスク優先順位の決定

## ステータス
Accepted

## コンテキスト
バックログに着手可能なタスクが複数あり、どの順序で進めるか決定が必要。

着手可能なタスク:
- T-607 (QA): セキュリティ修正検証テスト
- T-505 (QA): 認証フローE2Eテスト
- T-573 (QA): 統合テスト
- T-572 (DEV): 本番構成設定
- T-621 (PM): 可観測性運用ガイド
- T-630 (PM): ランディングページ仕様策定

## 選択肢

### A. テスト系タスク (T-607, T-505, T-573) を優先
- セキュリティ・品質の担保が先
- 本番デプロイ前に品質を固める

### B. 本番構成 (T-572) を優先
- デプロイを早期に実現できる
- ただし本番 D1/Queue の実 ID が手元にない

### C. ランディングページ (T-630 → T-631) を優先
- ユーザー向け機能の完成
- ただしMVP機能には直接影響しない

## 決定
**選択肢 A** を採用。T-607（セキュリティテスト）と T-621（可観測性ガイド）を並行して着手する。

## 理由
1. T-572 は本番 Cloudflare リソースの実 ID が必要で、現時点では手元にない
2. セキュリティ修正 (T-601〜606) は実装済みだがテスト未実施。本番公開前に検証が必須
3. T-621 はドキュメント作業で開発と並行可能
4. T-607 完了後、T-505 → T-573 の順でテストカバレッジを拡大する
