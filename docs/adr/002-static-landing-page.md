# ADR-002: トップページを静的ランディングページに変更

## ステータス
Accepted

## コンテキスト
従来のトップページ（`/`）は以下の動作をしていた:
1. クライアント JS で `GET /api/me` を呼び出し
2. 認証済み → `/stocks` にリダイレクト
3. 未認証 → ログイン案内を表示

この設計には以下の課題があった:
- **初回表示が遅い**: `/api/me` の応答を待つ間ローディングスピナーが表示される
- **SEO・共有不利**: クローラーやリンク共有時にコンテンツが見えない
- **サービス紹介が不十分**: 未認証ユーザーに対してサービスの価値を十分に伝えられていない

## 選択肢

### A. 静的ランディングページ（認証チェックなし）
- `/` は完全な静的 HTML（API 呼び出しゼロ）
- サービス概要・対応プロバイダ・使い方を表示
- CTA ボタンで `/login`（ログイン）と `/stocks`（一覧）へ誘導
- 認証済みユーザーの自動リダイレクトは行わない

### B. 従来通り認証チェック + リダイレクト
- 認証済みユーザーは自動的に `/stocks` へ
- 未認証ユーザーにはログインボタンのみ

### C. SSR でサーバーサイド判定
- Cookie をサーバーサイドで検証してリダイレクト
- Cloudflare Pages (static output) では不可

## 決定
**選択肢 A** を採用。

## 理由
1. **パフォーマンス**: API 呼び出しゼロで即座にコンテンツが表示される
2. **サービス訴求**: 初回訪問ユーザーにサービスの価値を伝えられる
3. **認証済みユーザーへの影響は軽微**: `/stocks` へのリンクがあるため 1 クリックで一覧に遷移可能
4. **静的ホスティングとの相性**: Cloudflare Pages の static output モードで最も自然な構成
5. **実装がシンプル**: クライアント JS の認証判定ロジックが不要になる

## 影響
- `src/pages/index.astro` から `<script>` タグと `/api/me` 呼び出しを完全除去
- E2E テストの「認証済みで `/` → `/stocks` リダイレクト」テストを除去し、ランディングページ表示テストに置き換え
