# 本番構成設定

wrangler.toml と関連設定を本番環境用に整備してください。

## 前提

- docs/deploy-checklist.md（T-571 で作成）の手順に従う
- 現在の wrangler.toml には開発用の設定がある

## やること

1. wrangler.toml を本番対応に更新する:
   - D1 の本番 database_id を設定する（または環境別設定）
   - Queue の本番設定を追加する
   - /api/* → Worker のルーティング設定

2. 環境別設定（dev / production）を整理する:
   - [env.production] セクションの追加（必要に応じて）
   - 本番 Secrets の一覧をコメントで記載する

3. `npm test` で全テストがパスすることを確認する

4. 変更をコミットする
