# リファクタ安全網テスト追加

コンポーネント抽出リファクタリングの前に、現在の動作を保護する安全網テストを追加してください。

## 前提

- T-200 で定義されたコンポーネント境界に基づき、リファクタ前に既存機能が壊れないことを保証するテストを書く
- docs/component-plan.md が存在する場合はそれを参照する

## やること

1. 現在の画面・APIの動作を確認する

2. 以下の安全網テスト（スナップショット/振る舞いテスト）を追加する:
   - ホームページが正しくレンダリングされること
   - ログインページが正しくレンダリングされること
   - ストック一覧ページが認証チェックを行うこと
   - `/api/health` が正常レスポンスを返すこと（既存テストで網羅済みなら追加不要）
   - `/api/me` が認証済み/未認証で適切に応答すること

3. テストが正しく失敗または成功することを確認する
   - 既存実装に対するテストは成功するべき
   - 未実装機能のテストは失敗してよい（skip 等で明示する）

4. `npm test` で全テストがパスすることを確認する
