# oEmbedフェッチサービス実装

各プロバイダから oEmbed メタデータを取得するサービスを実装してください。

## 前提

- docs/oembed-spec.md（T-520 で作成）の仕様に従う
- T-524 の QA テストが先に作成されている前提（TDD）

## やること

1. docs/oembed-spec.md を読み、各プロバイダの oEmbed 仕様を把握する

2. worker/lib/oembed.ts を作成する:
   - `fetchOembedMetadata(provider: string, canonicalUrl: string): Promise<OembedResult>` を実装する
   - SpeakerDeck: oEmbed JSON エンドポイントから取得
   - Docswell: oEmbed JSON エンドポイントから取得
   - Google Slides: embed URL を構築（`https://docs.google.com/presentation/d/{id}/embed`）
   - 取得するフィールド: title, author_name, thumbnail_url, embed_url
   - エラーハンドリング: HTTP エラー、タイムアウト、パースエラー

3. `npm test` で全テストがパスすることを確認する

4. 変更をコミットする
