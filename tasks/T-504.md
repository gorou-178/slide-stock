# フロントエンドGoogle Sign-In統合

login.astro に Google Sign-In ボタンを統合し、ログインフローを実装してください。

## 前提

- docs/auth-spec.md の仕様に従う
- T-501 の POST /api/auth/callback が実装済みであること
- src/pages/login.astro に現在プレースホルダーがある

## やること

1. Google Identity Services (GIS) ライブラリを導入する:
   - `<script src="https://accounts.google.com/gsi/client">` を追加する
   - GOOGLE_CLIENT_ID を環境変数から取得する

2. src/pages/login.astro を更新する:
   - Google Sign-In ボタンを配置する
   - コールバックハンドラで ID Token を POST /api/auth/callback に送信する
   - 成功時: /stocks にリダイレクトする
   - 失敗時: エラーメッセージを表示する

3. `npm test` と `npm run test:e2e` で全テストがパスすることを確認する

4. 変更をコミットする
