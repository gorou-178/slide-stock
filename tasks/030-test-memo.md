# メモ機能のテスト作成

メモ API エンドポイントのテストを作成してください。

## やること

1. `PUT /stocks/:id/memo` のテストを作成する
   - メモを新規作成できること (200)
   - 既存メモを更新できること (200)
   - メモ本文が空の場合に 400 を返すこと
   - 他ユーザーのストックにメモを作成で 404 を返すこと
   - 存在しないストック ID で 404 を返すこと
   - 認証なしで 401 を返すこと
2. `GET /stocks/:id/memo` のテストを作成する
   - メモを取得できること (200)
   - メモが存在しない場合に空レスポンスを返すこと (200, memo_text: null)
   - 他ユーザーのストックのメモ取得で 404 を返すこと
3. メモとストックの整合性テスト
   - ストック削除時にメモも削除されること（022-test-stocks と連携）
   - 1つのストックに対して1つのメモのみ存在できること（UPSERT 動作）

## 注意

- テストはこの段階では失敗してよい（Dev が後で実装する）
