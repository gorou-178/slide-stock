# 本番デプロイチェックリスト策定

MVP を本番環境（slide-stock.gorou.dev）にデプロイするためのチェックリストを作成してください。

## 前提

- docs/architecture.md の技術構成を参照する
- T-400 で作成された本番運用設計を参照する
- 現在のデプロイ: Cloudflare Pages + Workers（slide-stock.gorou.dev）

## やること

1. docs/deploy-checklist.md を作成し、以下を定義する:

   ### 事前準備
   - Google Cloud Console での OIDC クライアント設定（Client ID, Redirect URI）
   - Cloudflare Secrets の投入（GOOGLE_CLIENT_ID, SESSION_SECRET）
   - D1 本番データベースの作成（wrangler d1 create）
   - D1 本番 DB へのマイグレーション実行
   - Cloudflare Queue の作成
   - wrangler.toml の本番 database_id, queue 名の設定

   ### デプロイ手順
   - Worker のデプロイ（npm run deploy:worker）
   - Pages のデプロイ（npm run deploy:pages）
   - DNS / ルーティング設定（/api/* → Worker）

   ### 動作確認
   - Google Login が動作する
   - スライド登録 → oEmbed 取得 → 一覧表示の全フロー
   - メモの作成・編集
   - ログアウト

2. 必要に応じて tasks/TASKS.md に補足タスクを追加する
