# E2E 統合テスト (全画面フロー)

MVP 全体を通した E2E 統合テストを作成してください。

## やること

1. ユーザー登録〜ログインフローのテストを作成する
   - 初回アクセス → ログイン画面 → 認証 → 一覧画面
2. スライド登録〜閲覧フローのテストを作成する
   - URL 入力 → 登録 → pending 表示 → (メタデータ取得) → ready 表示
   - 各プロバイダ (SpeakerDeck, Docswell, Google Slides) で 1 件ずつ
3. メモ機能のフローテストを作成する
   - スライド詳細 → メモ入力 → 保存 → 再表示で確認
   - メモ更新 → 一覧画面でプレビューに反映
4. 削除フローのテストを作成する
   - スライド削除 → 一覧から消えること → メモも消えていること
5. ネガティブケースのテストを作成する
   - 不正 URL の入力
   - 重複 URL の入力
   - 未ログインでのアクセス

## 注意

- 認証バイパスを使用してテストする
- oEmbed の外部 API 呼び出しはモックする（E2E でも外部依存を避ける）
