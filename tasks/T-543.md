# Memo APIユニットテスト

Memo API の自動テストを作成してください。

## 前提

- docs/memo-api-spec.md の仕様に基づいてテストを設計する
- テスト対象: PUT/GET /api/stocks/:id/memo
- src/test/seed.ts のテストデータを使用する

## やること

1. test/worker/memo.test.ts を作成する:

   ### PUT /api/stocks/:id/memo
   - 正常系（新規作成）: → 200 + memo オブジェクト
   - 正常系（更新）: 既存メモを上書き → 200
   - 存在しない stock: → 404
   - 他ユーザーの stock: → 404
   - バリデーション: memo_text が空 → 仕様に応じた挙動
   - バリデーション: memo_text が最大文字数超過 → 400
   - 認証なし: → 401

   ### GET /api/stocks/:id/memo
   - 正常系: → 200 + memo オブジェクト
   - メモ未作成: → 404
   - 存在しない stock: → 404
   - 他ユーザーの stock: → 404
   - 認証なし: → 401

2. テストが正しく「失敗する」ことを確認する（実装前の場合）

3. テストコードをコミットする
