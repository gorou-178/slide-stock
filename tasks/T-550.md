# フロントエンドUI仕様策定

MVP のフロントエンド UI 構成とコンポーネント設計を定義してください。

## 前提

- CLAUDE.md の画面構成（セクション7）を参照する
- docs/component-plan.md（既存の場合）を参照する
- 現在の実装: src/pages/ に index.astro, login.astro, stocks.astro のプレースホルダーがある

## やること

1. CLAUDE.md のMVP画面構成を確認する:
   - ログイン画面: Google Login ボタン
   - 一覧画面: URL入力フォーム + ストック一覧（タイトル、サムネイル、メモ、元リンク、embed）
   - 詳細画面: embed表示 + メモ編集

2. docs/ui-spec.md を作成し、以下を定義する:
   - ページ構成（URL ルーティング）
   - コンポーネント階層:
     - BaseLayout（共通レイアウト: ナビバー、フッター）
     - URLInputForm（URL入力フォーム）
     - StockCard（ストックカード: タイトル、サムネイル、メモ抜粋、リンク）
     - EmbedViewer（oEmbed iframe 表示）
     - MemoEditor（メモ編集テキストエリア）
     - 等
   - 状態管理: ローディング、空状態、エラー状態の表示内容
   - レスポンシブ対応方針
   - CSS/スタイリング方針（素の CSS / フレームワーク）
   - アクセシビリティ要件（WAI-ARIA、キーボード操作、セマンティック HTML）

3. QA/DEV タスク（T-551〜T-558）が実行可能になるよう、仕様を具体的に記述する
