# プロバイダ検出仕様策定

SpeakerDeck / Docswell / Google Slides の URL パターンと正規化ルールを定義してください。

## 前提

- CLAUDE.md の対応プロバイダ（SpeakerDeck / Docswell / Google Slides）を参照する
- src/test/seed.ts にサンプル URL がある

## やること

1. 各プロバイダの公開スライド URL パターンを調査する:
   - SpeakerDeck: `https://speakerdeck.com/{user}/{slug}` 等
   - Docswell: `https://www.docswell.com/s/{user}/{slug}` 等
   - Google Slides: `https://docs.google.com/presentation/d/{id}/...` 等

2. docs/provider-spec.md を作成し、以下を定義する:
   - 各プロバイダの URL パターン（正規表現）
   - canonical URL への正規化ルール（クエリパラメータ除去、www 有無、末尾スラッシュ等）
   - provider 識別子（"speakerdeck" / "docswell" / "google_slides"）
   - バリデーションエラーメッセージ（未対応 URL、無効な形式 等）
   - エッジケース（短縮 URL、embed URL、edit URL 等）

3. QA/DEV タスク（T-511, T-512）が実行可能になるよう、仕様を具体的に記述する
