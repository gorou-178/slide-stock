# 共通UI抽出（TDD段階移行）

T-200 で定義されたコンポーネント境界に従い、共通UIコンポーネントを抽出してください。

## 前提

- docs/component-plan.md のコンポーネント定義に従う
- T-210 で追加された安全網テストが全てパスする状態を維持する

## やること

1. docs/component-plan.md を読み、移行順序を確認する

2. 移行順序に従い、以下の流れでコンポーネントを抽出する:
   - ページファイルから共通部分を Astro コンポーネント（`src/components/`）に切り出す
   - 各コンポーネントは単一責務を持つ
   - コンポーネント間のインターフェース（props）を明確にする

3. 各ステップで以下を確認:
   - `npm test` でユニットテストがパスすること
   - `npm run build` でビルドが通ること

4. 抽出完了後、コミットする
   - コミットはコンポーネント単位で小さく区切る
   - 例: `refactor(ui): extract NavBar component`
