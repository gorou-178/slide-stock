# Secrets一覧確定

本番環境で必要な Secrets（秘匿情報）の一覧を確定してください。

## やること

1. コードベースを確認し、環境変数・Secrets の使用箇所を洗い出す:
   - `.dev.vars` の内容
   - `wrangler.toml` の vars / secrets 参照
   - worker/ 内での `env.*` アクセス

2. 以下の Secrets 一覧を docs/secrets.md に記述する:

### Cloudflare Workers Secrets（`wrangler secret put` で設定）
- `GOOGLE_CLIENT_ID` — Google OIDC クライアントID
- `GOOGLE_CLIENT_SECRET` — Google OIDC クライアントシークレット
- `SESSION_SECRET` — セッショントークン署名用シークレット
- その他必要なもの

### GitHub Actions Secrets
- `CLOUDFLARE_API_TOKEN` — Wrangler デプロイ用
- `CLOUDFLARE_ACCOUNT_ID` — Cloudflare アカウントID

3. 各 Secret の:
   - 用途
   - 設定方法
   - ローテーション方針（必要な場合）

4. `.dev.vars` と本番 Secrets の対応表を作成する
