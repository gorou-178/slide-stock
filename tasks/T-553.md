# URL送信フロー実装

ストック一覧画面の URL 入力フォームからスライドを登録するフローを実装してください。

## 前提

- T-531（POST /api/stocks）が実装済みであること
- T-552（ストック一覧画面）が実装済みであること

## やること

1. URL 入力フォームの送信ハンドラを実装する:
   - URL のクライアントサイドバリデーション
   - POST /api/stocks にリクエストを送信する
   - 送信中のローディング表示
   - 成功時: pending 状態のストックカードを一覧に追加する
   - エラー時: エラーメッセージを表示する（未対応 URL、重複 等）

2. `npm run test:e2e` で全テストがパスすることを確認する

3. 変更をコミットする
