# スライドストック CRUD のテスト作成

スライドストックの API エンドポイントに対するテストを作成してください。

## やること

1. `POST /stocks` のテストを作成する
   - 有効な URL で stock が作成されること (201)
   - レスポンスに stock の情報が含まれること
   - status が `pending` で作成されること
   - Queue にメッセージが enqueue されること
   - 認証なしで 401 を返すこと
   - 不正な URL で 400 を返すこと
   - 未対応プロバイダで 400 を返すこと
   - 重複 URL で 409 を返すこと
2. `GET /stocks` のテストを作成する
   - 認証ユーザーのストック一覧を返すこと
   - 他ユーザーのストックが含まれないこと
   - created_at 降順でソートされていること
   - 空一覧の場合に空配列を返すこと
3. `GET /stocks/:id` のテストを作成する
   - 指定 ID のストック情報を返すこと
   - 他ユーザーのストックにアクセスで 404 を返すこと
   - 存在しない ID で 404 を返すこと
4. `DELETE /stocks/:id` のテストを作成する
   - ストックが削除されること (204)
   - 関連するメモも削除されること
   - 他ユーザーのストック削除で 404 を返すこと

## 注意

- テストはこの段階では失敗してよい（Dev が後で実装する）
- Queue の enqueue はモックする
