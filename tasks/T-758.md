# T-758 — [設計] メモ機能改善設計

## 背景

`worker/handlers/memo.ts` のメモ機能に以下の改善要望がある:

- 上限が 10,000 文字（L12: `MEMO_MAX_LENGTH = 10000`）→ 200文字に変更
- 改行の扱いが未定義 → 改行を許容し、LFに統一
- トリムがバリデーション用のみ → 保存時に前後トリム

## 現状コード（memo.ts）

- L12: `const MEMO_MAX_LENGTH = 10000`
- L58: `memoText.trim() === ""` で空チェック
- L62-68: `[...memoText].length > MEMO_MAX_LENGTH` で文字数チェック（Unicode対応）
- 改行の正規化処理なし
- 保存時のトリム処理なし（バリデーション後の値をそのまま INSERT/UPDATE）

## 設計すべき内容

### 1. 文字数上限の変更

- 10,000 → 200 文字
- 既存データで 200 文字を超えるメモがある場合の扱い:
  - 個人ツールかつ初期段階のため、超過データは存在しない前提でよいか確認
  - マイグレーションでの切り詰めは不要と判断してよいか

### 2. 改行処理

- 改行を許容する（複数行メモ対応）
- 保存時に CRLF → LF に正規化: `memoText.replace(/\r\n/g, '\n').replace(/\r/g, '\n')`
- 連続改行の制限は設けるか？（例: 3連続以上を2連続に圧縮）
- フロント表示時: `<br>` 化 or CSS `white-space: pre-line`

### 3. 前後トリム

- 保存時に `memoText.trim()` を適用
- バリデーション（空チェック・文字数チェック）はトリム後の値で実施

### 4. フロント側の変更

- 入力: `<input>` → `<textarea>` への変更が必要か
- 表示: memo_text の改行を `<br>` に変換 or CSS で対応
- 文字数カウンター表示（200文字制限の可視化）
