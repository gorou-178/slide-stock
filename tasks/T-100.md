# CI/CD方針とDoDを plan.md に明文化

docs/ci-cd-plan.md を作成し、CI/CD方針とDefinition of Done (DoD) を明文化してください。

## やること

1. CLAUDE.md と docs/architecture.md を読み、現在のプロジェクト構成を確認する
2. 以下の内容を docs/ci-cd-plan.md に記述する:

### CI（継続的インテグレーション）
- PR作成時にどのテストを実行するか（vitest, playwright, TypeScript型チェック, ビルド確認）
- 実行コマンド一覧（`npm test`, `npm run test:e2e`, `npx tsc --noEmit`, `npm run build`）
- テストが落ちた場合のマージポリシー

### CD（継続的デプロイ）
- main マージ後の自動デプロイフロー
- Cloudflare Pages (フロントエンド) と Workers (API) のデプロイ手順
- Secrets / 環境変数の管理方針

### DoD（Definition of Done）
- タスク完了の基準（テストパス、型エラーなし、ビルド成功、レビュー通過）
- PM / QA / DEV 各ロールの完了条件

3. tasks/TASKS.md への新タスク追加は不要（既に定義済み）
