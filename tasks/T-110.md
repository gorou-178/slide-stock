# 現状テスト棚卸しとCI実行コマンド定義

現在のテストコードを棚卸しし、CI で実行すべきコマンドを整理してください。

## やること

1. 既存のテストファイルを全て確認する:
   - `test/worker/health.test.ts`
   - `worker/middleware/test-auth-bypass.test.ts`
   - `e2e/home.test.ts`
   - `e2e/auth-bypass.test.ts`

2. 各テストが何を検証しているか簡潔にまとめる

3. 以下のコマンドを実行し、全てパスすることを確認する:
   - `npm test` (vitest ユニットテスト)
   - `npx tsc --noEmit` (TypeScript 型チェック)
   - `npm run build` (Astro ビルド)

4. 結果を docs/test-inventory.md に記録する:
   - テストファイル一覧と検証内容
   - CI で実行すべきコマンド一覧（実行順序含む）
   - 現時点で不足しているテストの概要（実装は別タスク）

5. 失敗するテストがあれば原因を特定し、修正する（テストコード側の問題のみ）
