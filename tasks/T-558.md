# フロントエンドE2Eテスト

フロントエンドの主要フローを検証する E2E テストを作成してください。

## 前提

- docs/ui-spec.md の仕様に基づいてテストを設計する
- Playwright を使用する
- e2e/fixtures/index.ts の authenticatedPage フィクスチャを使用する

## やること

1. E2E テストを作成する:

   ### ログインフロー (e2e/login-flow.test.ts)
   - 未認証で /stocks → /login にリダイレクト
   - ログインボタンが表示される
   - 認証済みで / → /stocks にリダイレクト

   ### ストック一覧 (e2e/stocks-list.test.ts)
   - 一覧ページが表示される
   - URL 入力フォームが表示される
   - ストックカードが正しく表示される（タイトル、サムネイル、メモ）
   - 空状態メッセージの表示
   - 元スライドへのリンクが正しい

   ### URL 登録フロー (e2e/stock-create.test.ts)
   - URL を入力して送信 → ストックが一覧に追加される
   - 無効な URL → エラーメッセージ表示

   ### ストック詳細 (e2e/stock-detail.test.ts)
   - 詳細ページが表示される
   - embed iframe が表示される
   - メモを編集して保存 → 反映される
   - 削除ボタン → 確認後一覧にリダイレクト

   ### アクセシビリティ
   - 各ページで基本的なアクセシビリティチェック

2. テストが正しく「失敗する」ことを確認する（実装前の場合）

3. テストコードをコミットする
