# E2E強化

既存の E2E テスト（Playwright）を拡充し、MVP の主要フローをカバーしてください。

## 前提

- 既存の E2E テスト:
  - `e2e/home.test.ts` — ホームページの基本表示
  - `e2e/auth-bypass.test.ts` — テスト認証フローの確認
- MVP の主要機能が実装済みであること

## やること

1. 以下のユーザーフローを E2E テストでカバーする:

### 認証フロー
- ログインページからの認証（テストモード）
- 認証後のリダイレクト
- 未認証でのアクセス制御

### スライド登録フロー
- URL 入力フォームからスライドを登録できること
- 不正な URL でエラーが表示されること
- 登録後に一覧に表示されること

### 一覧表示
- ストック一覧が表示されること
- 各スライドのタイトル・サムネイルが表示されること
- 元スライドへのリンクが機能すること

### メモ機能
- メモの作成・編集ができること
- メモが保存され、再表示時に反映されること

### スライド削除
- スライドを削除できること
- 削除後に一覧から消えること

2. `npm run test:e2e` で全テストが実行できることを確認する
3. テストが失敗する場合は skip して理由をコメントに残す（実装が未完了の場合）
