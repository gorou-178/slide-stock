# コンポーネント境界と移行順を定義

現在のフロントエンド（src/pages/）はページファイルにロジックが直書きされている。
コンポーネント分割の境界と、移行の優先順位を定義してください。

## やること

1. 現在の src/pages/ のコードを確認する:
   - `index.astro`
   - `login.astro`
   - `stocks.astro`

2. CLAUDE.md の画面構成（ログイン画面・一覧画面・詳細画面）を元に、必要なコンポーネントを洗い出す:
   - 例: `SlideCard`, `UrlInputForm`, `MemoEditor`, `EmbedViewer`, `NavBar`, `LoginButton` 等

3. docs/component-plan.md を作成し、以下を記述する:
   - コンポーネント一覧（名前・責務・配置先）
   - 依存関係（どのページがどのコンポーネントを使うか）
   - 移行順序（影響が小さく独立性の高いものから）
   - TDD の流れ: QA がテスト → DEV が実装の順番を明示する

4. 対応する QA/DEV タスクが必要であれば tasks/TASKS.md に追加する
