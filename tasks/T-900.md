# 監視方針（Sentry等）

本番運用における監視・エラートラッキングの方針を定義してください。

## やること

1. 以下の観点で監視方針を docs/monitoring-plan.md に記述する:

### エラートラッキング
- Sentry / Cloudflare Workers Analytics / その他の選択肢
- コスト（無料枠で十分か）
- Workers への組み込み方法

### ログ
- Cloudflare Workers のログ確認方法（`wrangler tail`）
- 構造化ログの方針
- ログレベル（error / warn / info）

### アラート
- 致命的エラーの通知方法
- デプロイ失敗時の通知（GitHub Actions で対応済みなら不要）

### ヘルスチェック
- `/api/health` エンドポイントの活用
- 外部監視サービスの必要性

2. MVP では最小限の監視で十分。過剰な導入は避ける
3. 具体的な導入が必要であれば別タスクを tasks/TASKS.md に追加する
