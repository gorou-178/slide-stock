# T-750 — [設計] Cookieセキュリティ強化設計

## 背景

`worker/handlers/auth.ts` のCookie設定に以下の問題がある:

1. **Cookie名に `__Host-` prefix がない**: `auth_state`, `session` → セキュリティ上 `__Host-` prefix を付与すべき
2. **Path=/api 固定**: SSR統合後、ページ側でセッション参照が必要になる可能性がある
3. **Max-Age ハードコード**: `auth_state=300`, `session=604800` がリテラル値

## 現状コード（auth.ts）

- L114: `auth_state` Cookie — `Max-Age=300; Path=/api; HttpOnly; Secure; SameSite=Lax`
- L212: `session` Cookie — `Max-Age=604800; Path=/api; HttpOnly; Secure; SameSite=Lax`
- L216, L237: Cookie削除時も同パス

## 設計すべき内容

### 1. `__Host-` prefix 付与

- `auth_state` → `__Host-auth_state`
- `session` → `__Host-session`
- `__Host-` prefix の要件: `Secure`, `Path=/`, Domain属性なし
- **注意**: `__Host-` は `Path=/` を要求するため、現在の `Path=/api` とは両立しない → Path の見直しが必須

### 2. Cookie Path 見直し

- 現状: `Path=/api`（APIエンドポイントにのみCookie送信）
- SSR統合済み（T-701）でAstro API Routesを使用中
- 検討事項:
  - ページ側でセッション情報が必要か（SSRでユーザー名表示等）
  - `Path=/` にした場合のセキュリティ影響
  - `__Host-` prefix 採用なら `Path=/` 必須

### 3. Max-Age 環境変数化

- `SESSION_MAX_AGE` と `AUTH_STATE_MAX_AGE` を Env から取得
- デフォルト値はフォールバックとして維持
- `wrangler.toml` の `[vars]` セクションに追加
